MariaDB Source → Platform Binary

Visual representation of how MariaDB source code becomes platform-specific binaries.

1. MariaDB Source Code

Core database engine written mainly in C/C++ including parser, optimizer, storage engines, replication, plugins, and networking.

2. Build System

CMake configures the project, detects compiler, architecture, operating system, and required libraries.

3. Compilation

Compiler converts source files into object files using architecture-specific instruction generation.

4. Linking

Object files are linked into executable binaries, shared libraries, and platform runtime packages.

5. Platform-Specific Outputs

Different CPU architectures produce different machine code binaries.

x86_64 / AMD64

  • Intel and AMD processors
  • 64-bit x86 instructions
  • ELF binaries on Linux
  • High-performance servers

ARM64 / AArch64

  • Apple Silicon
  • AWS Graviton
  • ARM server infrastructure
  • Different opcode generation

ARM32

  • Embedded systems
  • 32-bit ARM instruction set
  • Low-power deployments
  • Smaller address space

Windows PE

  • Built using MSVC or MinGW
  • PE/COFF executable format
  • DLL runtime linkage
  • Windows service support
