Rate Limiting Setup

We need admin rights to OS level configuration for this facility.

Step 1: Create Rate Limiting File

sudo nano /usr/local/bin/rate_limit.sh

Step 2: Paste Following Script

#!/bin/bash

# Flush old rules
iptables -F

# Default policies (safer)
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# Allow established connections
iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT

# Allow localhost
iptables -A INPUT -i lo -j ACCEPT

# ✅ Allow your trusted/local IP (NO LIMIT)
iptables -A INPUT -s 192.168.22.35 -j ACCEPT

# ✅ Rate limit ONLY other IPs (new connections) and 100 bursts in 5sec
iptables -A INPUT -p tcp -m conntrack --ctstate NEW \
  -m hashlimit \
  --hashlimit 100/5sec \
  --hashlimit-burst 100 \
  --hashlimit-mode srcip \
  --hashlimit-name conn_limit \
  -j ACCEPT

# Allow required ports (for those who passed rate limit)
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Custom ports
iptables -A INPUT -p tcp --dport 7299 -j ACCEPT
iptables -A INPUT -p tcp --dport 7298 -j ACCEPT
iptables -A INPUT -p tcp --dport 7777 -j ACCEPT
iptables -A INPUT -p tcp --dport 8081 -j ACCEPT
iptables -A INPUT -p tcp --dport 8080 -j ACCEPT
iptables -A INPUT -p tcp --dport 8585 -j ACCEPT

# Drop everything else
iptables -A INPUT -j DROP

Step 3: Make File Executable

sudo chmod +x /usr/local/bin/rate_limit.sh

Step 4: Create Auto Start Service

sudo nano /etc/systemd/system/rate_limit.service

Step 5: Paste Service File Content

[Unit]
Description=Rate Limit Firewall Rules
After=network.target

[Service]
Type=oneshot
ExecStart=/usr/local/bin/rate_limit.sh
RemainAfterExit=true

[Install]
WantedBy=multi-user.target

Step 6: Enable and Start Service

sudo systemctl daemon-reexec
sudo systemctl daemon-reload
sudo systemctl enable rate_limit.service
sudo systemctl start rate_limit.service

Step 7: Stop Service

sudo systemctl stop rate_limit.service
sudo systemctl disable rate_limit.service

Step 8: Check Service Status

sudo systemctl status rate_limit.service
