Netplan and Wifi Watchdog setup

We need admin rights to OS level configuration for this facility.

Check Internet Connection Type

To check internet is connecting via wifi or ethernet:

ip -o -4 route show to default | awk '{print $5}'

Check Current Device IP Address

To check current device ip address:

hostname -I | awk '{print $1}'

Check Default Route or Gateway

To check default route or gateway:

ip route | grep default | awk '{print $3}'

Check First DNS Server Address

To check first dns server address:

awk '/^nameserver/ {print $2; exit}' /etc/resolv.conf

Go to Netplan Directory

Go to following directory:

cd /etc/netplan

Remove Default Cloud Init File

If you find 50-cloud-init.yaml, remove it:

ls

sudo rm 50-cloud-init.yaml

Create Custom Netplan Configuration

Create custom configuration:

sudo nano 01-netcfg.yaml

Paste Netplan Configuration

Assume:
Device IP: 192.168.22.30
Default Route or Gateway: 192.168.22.21
DNS: 77.88.8.1
Alternate DNS: 77.88.8.88
Wifi Name: Wonder
Password: abcdefgh 

network:
  version: 2
  renderer: networkd

  ethernets:
    eth0:
      dhcp4: no
      optional: true
      addresses:
        - 192.168.22.30/24
      routes:
        - to: default
          via: 192.168.22.21
          metric: 100
      nameservers:
        addresses:
          - 77.88.8.1
          - 77.88.8.88

  wifis:
    wlan0:
      dhcp4: no
      optional: true
      addresses:
        - 192.168.22.30/24
      routes:
        - to: default
          via: 192.168.22.21
          metric: 600
      nameservers:
        addresses:
          - 77.88.8.1
          - 77.88.8.88
      access-points:
        "Wonder":
          password: "abcdefgh"

Secure Netplan File

To make the file executable:

sudo chmod 600 /etc/netplan/01-netcfg.yaml

Safely Test Netplan Configuration

This is important and do not ignore it, unless you may lock your system internet. 

sudo netplan try

Apply Netplan Configuration

If connection is ok, then apply following:

sudo netplan apply

Create WiFi Watchdog Script

To create a wifi connectivity checker dog: 

sudo nano /usr/local/bin/wifi-watchdog.sh

Paste WiFi Watchdog Script

Paste following under above wifi context:

#!/bin/bash

IFACE="wlan0"
GATEWAY="192.168.22.21"
TARGET="77.88.8.1"

LOG_FILE="/var/log/wifi-watchdog.log"

log() {
    echo "$(date): $1" >> "$LOG_FILE"
}

check_ping() {
    ping -c 3 -W 2 -I "$IFACE" "$1" > /dev/null 2>&1
}

# ---- STEP 1: Check connectivity ----
check_ping "$GATEWAY"
GW_OK=$?

check_ping "$TARGET"
NET_OK=$?

# ---- CASE 1: Everything OK ----
if [ $GW_OK -eq 0 ] && [ $NET_OK -eq 0 ]; then
    log "Connection OK"
    exit 0
fi

log "Warning: connectivity issue detected (gw=$GW_OK net=$NET_OK)"

# ---- STEP 2: Soft recovery (NO DROP) ----
log "Attempting soft recovery via networkd..."

networkctl reconfigure "$IFACE" 2>/dev/null

sleep 5

# Re-check after soft recovery
check_ping "$GATEWAY"
GW_OK2=$?

check_ping "$TARGET"
NET_OK2=$?

if [ $GW_OK2 -eq 0 ] && [ $NET_OK2 -eq 0 ]; then
    log "Soft recovery succeeded"
    exit 0
fi

# ---- STEP 3: Restart Wi-Fi authentication layer ----
log "Soft recovery failed. Restarting wpa_supplicant..."

systemctl restart wpa_supplicant@"$IFACE".service 2>/dev/null

sleep 8

# Re-check again
check_ping "$TARGET"
NET_OK3=$?

if [ $NET_OK3 -eq 0 ]; then
    log "Recovery successful after wpa_supplicant restart"
    exit 0
fi

# ---- STEP 4: LAST RESORT (still controlled) ----
log "Hard recovery required"

ip link set "$IFACE" down
sleep 2
ip link set "$IFACE" up

netplan apply

sleep 10

check_ping "$TARGET"
if [ $? -eq 0 ]; then
    log "Hard recovery succeeded"
else
    log "WARNING: Full recovery failed"
fi

Make Watchdog Executable

To make executable file:

sudo chmod +x /usr/local/bin/wifi-watchdog.sh

Create System Boot Service for Watchdog

To make a system boot service:

sudo nano /etc/systemd/system/wifi-watchdog.service

Paste Watchdog Service Content

Paste following:

[Unit]
Description=WiFi Connectivity Checker

[Service]
Type=oneshot
ExecStart=/usr/local/bin/wifi-watchdog.sh

Create Watchdog Timer

To attach a timer that periodically checks connectivity:

sudo nano /etc/systemd/system/wifi-watchdog.timer

Paste WiFi Watchdog Timer Content

Paste following:

[Unit]
Description=Run WiFi check every 20 seconds

[Timer]
OnBootSec=30
OnUnitActiveSec=20
Unit=wifi-watchdog.service

[Install]
WantedBy=timers.target

Enable Watchdog Timer

Then following commands are system setups:

sudo systemctl daemon-reexec
sudo systemctl daemon-reload
sudo systemctl enable --now wifi-watchdog.timer
